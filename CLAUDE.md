# Правила проекта jtprog.ru

## Структура повествования

Любой связный текст в этом проекте (посты, заметки, доклады, README) строится как история с внятной дугой, а не как набор фактов. Перед письмом — проверь скелет по синопсису (о чём / кто / цель / зачем / препятствия), после черновика — проверь поток по пирамиде Фрейтага (экспозиция → нарастание → кульминация → спад → развязка). Это ориентир, а не догма: короткую заметку-рецепт можно осознанно свернуть до пары актов.

Полное правило: [.agents/rules/narrative-structure.md](.agents/rules/narrative-structure.md). Сверяйся с ним всегда, когда пишешь или редактируешь текст — особенно при работе через навыки `jtprogru-blog-style` и `content-humanizer`.
